Terminology
Name Description
Selector The beginning of the ruleset used to target the element that will be styled.
Declaration Block The code in-between (and including) the curly braces { } that contains the CSS declaration(s).
Declaration The group name for a property and value pair that applies a style to the selected element.
Property The first part of the declaration that signifies what visual characteristic of the element is to be modified.
Value The second part of the declaration that signifies the value of the property.
Specificity The order by which the browser decides which CSS styles will be displayed in the browser. In order of least to most specific: type, class, ID. An element that has an ID and class style will display the ID style instead of the class style. Additionally, a style with 2 type selectors would be displayed instead of a style with 1 type selector, and so on.
Linking CSS to HTML <link> within the <head> of an HTML file. Requires href="addressofcssfile", and rel="stylesheet". If .css file is stored in the same directory, use a relative path.
Selectors
Example Name Description
<p style="color: red;">Hello, World!</p> Inline (HTML) Within an HTML element's opening tag, adding the style attribute allows you to insert CSS code inline with your HTML.
p {} Type Matches the HTML element in the document. Do not use angle brackets when selecting elements in CSS. The type selector is also referred to as a tag name or element selector.
* {} Universal Selects all elements of any type.
.paints {} Class In HTML, an element is given the attribute class="classname" in the opening tag. In CSS, the classname is written as shown in the example (with a . before its name). One of the most common ways to select an element when working with HTML and CSS. Meant to be reused over many elements.
.paints {} .bold {} Multiple Classes In HTML, the class attribute will contain multiple class names (class="paints bold"). In CSS, the classes will each have their own entries.
#big-title {} ID In HTML, the id attribute will be applied to the element in the opening tag (id="big-title"). In CSS, the ID is written with a # before its name. If an HTML element needs to be styled uniquely, give it an id attribute. An ID can only be used once per page, and overrides the styles of types and classes.
[href] {} Attribute Used to target HTML elements that already contain an attribute. Targets all elements with the given attribute. In CSS, the attribute must be surrounded with square brackets [].
img[src*="winter"] {} Type and/or Attribute Selects an element with an attribute that contains any instance of the specified value. type[attribute*="value"].
p:hover {} Pseudo-class Changes text (gives elements a different state) in response to user interaction, site navigation, and position in the document tree. Can be attached to any selector. Includes: :link :visited :hover :active :focus :disabled.
.main-list li {} Descendant Combinator Selecting elements that are nested within other elements (descendants). Nesting occurs when an element, and its opening and closing tags, are between the opening and closing tags of another element. In the example, <li> is nested between an element that has the attribute class="main-list".
h1.special {} Chaining Assigning 2+ CSS selectors to an HTML element at the same time (combining selectors). This example allows you to select only the h1 elements with a class of special.
h1, .menu {} Multiple Selectors Assigning CSS style(s) to multiple selectors at the same time (unrelated HTML elements).
.attribute a::after {} Pseudoselector Selects the space that occurs ::before or ::after the element.
Visual Rules
Example Name Description
font-family: arial; Font Family Changes the typeface, or the font of the text. When the name of a typeface contains multiple words, it's best to enclose it inside of quotes, e.g. "Courier New". Font must be installed on the user's computer or downloaded with the site. Web safe fonts appear the same across all browsers and operating systems. A font stack is a list of multiple similar-looking fonts (called fallback fonts). The browser will try to use the first font. If it can't, it will use the second, and finally the third. Serif fonts have extra details on the ends of each letter. Sans-Serif fonts do not have the extra details. These keyword values (serif or sans-serif) can be used as final fallback fonts if the browser is unable to support the other fonts in the font stack.
font-size: 18px; Font Size Changes the size of the text.
font-weight: bold; Font Weight How bold or thin text appears. Keyword values include: lighter, normal, bold, and bolder, all relative to the parent element. Numerical values range from 1-1000 (lightest to boldest). Common practice is to use increments of 100 (400 = normal, 700 = bold). Not all fonts can be assigned a numeric font weight. Not all numeric font weights are available to all fonts.
text-align: center; Text Align Aligns text relative to the parent element that holds it. Includes: left, center, right, and justify (spreads text across the width of the parent element).
color: blue; Color Styles an element's foreground color (the element itself).
background-color: black; Background Color Styles an element's background color (the space around the element).
opacity: 0.5; Opacity The measure of how transparent an element is. Scale of 0-1, using decimals (0 is invisible, 1 is fully visible).
background-image: url(“ ”); Background Image Changes the background of an element to display an image. Copy the path (absolute) and insert it between the “ “, use forward slashes.
color: blue !important; Important Overrides any style (no matter how specific). Changes all elements of that selector. Applied to specific declarations, instead of full rules. Should almost never be used.
content: ">"; Content Can be used to characterize the space between breadcrumbs. Includes: ">" "/" and "x".
vertical-align: super; Vertical Align Aligns the text vertically, within its line of text.
list-style: none; List Style Under the <ul> selector (or its class name), list style allows you to style, or even remove the bullets. Includes: none;.
The Box Model
Example Name Description
width: 240px; Width Changes the width of the content area. min-width and max-width ensure a minimum and maximum width for the element's box, respectively. When set in pixels, the box will be the same size on all devices. Height, width, and padding can also be set in %.
height: 80px; Height Changes the height of the content area. min-height and max-height ensure a minimum and maximum height for the element's box, respectively.
padding: 10px; Padding The amount of space between the content area and the border. This example puts 10 pixels of space between the content and the border on all 4 sides. Customize each side: padding-top, padding-bottom, padding-right, padding-left. Shorthand example: padding: 10px 5px 10px 5px; values are listed going clockwise around the box.
border: 3px solid coral; Border The thickness and style of the border surrounding the content area and padding. Can be set with specific: width (thickness of border; set in pixels (px), or thin, medium, thick), style (design of border; solid, several styles), and color. The default border is medium none color, where color is the current color of the element. Any aspect that is not indicated will be set to the default.
border-radius: 5px; Border Radius Modifies the corners of an element's border box. This example sets all 4 corners of the border box to a 5px radius. You can create a circular border by setting the height and width to the same value, and the radius to 50%.
margin: 0px auto; Margin The amount of space between the border and the outside edge of the element. Same rules apply to margin as padding. More common to see margin values set for a specific side of an element. Auto allows you to center content. In the example, the 0 sets the margins for the top and bottom of the box, and the auto centers the element left-right. To center an element, a width must be set for it. Margin collapse: top and bottom (vertical) margins can collapse, unlike padding. Whichever element has a larger vertical margin, that will set the distance between them vertically on the page. Left + right (horizontal) margins will be spaced equal to the sum of their margins.
overflow: scroll; Overflow Controls what happens to content that overflows, or spills outside of the box. hidden: content that overflows will be hidden from view. scroll: content that overflows will create a scrollbar to view it. visible: content that overflows will be displayed outside of the container element. This is the default value. Set overflow on a parent element to control the visibility of child elements.
visibility: visible; Visibility Adjusts the visibility of elements. hidden: hides the element (leaves empty space on webpage). visible: displays the element. collapse: collapses the element. display: none will remove the element from the webpage.
* {margin: 0; padding: 0;} Resetting Defaults All major web browsers have a default stylesheet that they use in the absence of an external stylesheet (known as user agent stylesheets). “user agent” = browser. The example is often the first CSS rule in an external stylesheet (resets all margin and padding).
* {box-sizing: border-box;} Box Sizing The Box Problem: borders, padding, height, and width all add together. This creates a problem, whereby the actual dimensions of the box change from 300px to something like 322px. This makes it hard to organize the elements on the webpage. box-sizing controls the type of box model that a browser should use when interpreting a web page. The default value of this property is content-box. However, border-box allows the height and width values to set the absolute values for the box. Under border-box, padding and borders will not add to the dimensions of the box.
Display and Positioning
Example Name Description
position: fixed; top: 50px; Position The default position of an element can be changed by setting its position property. static is the default value and does not need to be specified. relative allows you to position an element relative to its default static position. Must be accompanied by an offset property, or properties (top, bottom, left, right, specified in px, ems, or %). absolute means that all other elements will ignore the element, acting like it's not even there. Positioned relative to its closest positioned parent element (offset properties can be used here as well). The element will scroll with the rest of the document. fixed fixes an element's position on the web page regardless of user scrolling. Commonly used for navigation bars across the top of a web page. Uses offset properties as well. sticky keeps an element in the document flow as the user scrolls. Sticks to a specified position as the page is scrolled further. Uses offset properties as well.
z-index: 1; Z Index Controls how far back, or how far forward elements appear when they overlap. Controls the depth of content relative to other content. Accepts integer values (1, 2, 3) - defaults to 0 - higher #'s are more visible. Does NOT work with a static position.
display: block; Display All HTML elements have a default display value that controls whether it can share horizontal space with another element. inline makes the content box wrap tightly around the element, so that it only takes up the space needed to display the element (does not require a new line after each element). Height and width of these elements can't be specified in the CSS doc. inline is the default display for some elements such as <a> <strong> <em>. block fills the entire width of the page by default. Content is not displayed in the same line as surrounding content. height and width properties can be set (unless specified, they are the height necessary to display their content). block is the default display for elements such as <h1> <p> <div> <footer>. inline-block content can appear next to each other (does not start new line). Width and height can be adjusted. inline-block is the default display for elements such as <img>.
float: right; Float Moves an element as far left or as far right as possible in the container. Must have width specified. Commonly used for wrapping text around an image, and floating multiple elements. Other tools for this purpose include CSS grid and flexbox.
clear: both; Clear left; the left side of the element will not touch any other element within the same containing element. right; the right side of the element will not touch any other element within the same containing element. both; neither side of the element will touch any other element within the same containing element. none; the element can touch either side.
Color
Example Name Description
color: midnightblue; Named Colors English words that describe colors (keyword colors). ~140 colors. transparent; eliminates color opacity.
color: rgb(255, 255, 255); RGB Numeric values that describe a mix of red, green, and blue. ~256 * 256 * 256 = 16,777,216 colors. Each value represents a color component (red, green, and blue, respectively, from 0-255). Control opacity (how transparent the color is) with color: rgba(0, 0, 0, 0.1). The "a" stands for alpha, and is the 4th number in the series (value scale of 0-1, in decimals).
color: hsl(120, 70%, 80%); HSL Numeric values that describe a mix of hue, saturation, and lightness. The first number represents the hue, on a scale of 0-360. The number corresponds to an angle on a color wheel. The second number represents the saturation, or intensity/purity of the color, from 0-100%. The third number represents the lightness, from 0-100%. Control the opacity with hsla(0, 0, 0, 0.3). The 4th number is the alpha, which controls the opacity on a value scale of 0-1.
color: #8FBC8F; Hexadecimal A type of syntax. Colors created using this syntax are called hex colors. # followed by 3 or 6 characters (letters and numbers). Hexadecimal number system has 16 digits (0-15). 10-15 are represented by the letters A-F (capitalized or lowercase). If the first and second values are the same (e.g. AA), they can be represented by one value (e.g. A). This is only true if the third and fourth, as well as the fifth and sixth values are also the same. Hex numbers must be either 3 or 6 values in length. For opacity, add 1 digit to a 3-digit hex, or add 2 digits to a 6-digit hex. Ranges from 00 (transparent) - FF (opaque).
background-color: rgb(0, 0, 0); Background Color All of these color properties can be applied to both color: and background-color:.

Secondary Navigation
Example Name Description
Primary vs. Secondary The primary navigation system typically contains the most important links and buttons that need to be displayed on every single page of the site. Secondary navigation, or breadcrumb navigation, usually consists of a clickable list of pages or attributes that led to the current page. It can help users understand the extent of the site and also where they are currently.
3 Types Location: based on where you are with respect to the navigation structure of the site. Attribute: based on the attributes of the page or product that you are browsing. If you want to allow users to remove attributes, provide an (x) button or similar to indicate they can be removed. Path: based on a user's unique path through the site. On complex websites, abbreviations are common ( … > About > Register ). Adding these classes to a breadcrumb list will allow you to differentiate them in the browser.
<ul class="breadcrumb"><li><a href="shoes.html">Shoes</a></l> Breadcrumbs Usually displayed as a horizontal list of pages. In the header, left-aligned, and below any primary navigation. Takes up minimal space. Typically, they are separated with a “>” or a “/“ symbol. Not underlined. Displays clickability via :hover properties. Links to their own page.
.breadcrumb li+li::before {content: ">"} Pseudoselector Selects the space that occurs ::before or ::after the element. Useful for editing the appearance of the space between breadcrumb tabs.
Grids and Spacing
Example Name Description
Columns The vertical sections that span the width of a page. In web design, it’s common to see layouts with 12-16 columns, and sometimes as few as 3 columns. The number of columns is determined by your design, device, and user screen size. Websites may not have the same column layout throughout. Content can span multiple columns.
Gutters The negative space between each column (there is always 1 less gutter than the number of columns). There are vertical and horizontal gutters. Gutters ensure that columns (and rows) don’t run together. They are typically narrower in width than the columns. Content can’t spill into the gutter unless it appears in the next column.
Margins Appear on the left and right sides of a column-based grid. Margins ensure that the content of your webpage doesn’t match up to the edges of the browser window. Margins may vary depending on the width of the grid, browser window, or device.
Responsive Design A set of techniques that allow a website’s content to shift based on the device or screen size. Requires a different number of total columns based on screen size, in order to accommodate content and keep it from being squished. For example: a 12-column grid on a desktop may be a 4-column grid on a mobile device.
Whitespace The negative area surrounding content (may or may not be white). Whitespace makes a web page easier to view and navigate. There are 2 forms of whitespace: passive and active. Passive / micro / unconsidered whitespace is used to improve the aesthetics of the layout without guiding the user. Passive whitespace is commonly used within text elements. Passive whitespace can be edited by using the line-height and margin styles. Active / macro whitespace is intentional, and is used to improve the page structure in order to guide a user through the web page. Active whitespace is commonly used to separate a site’s elements.
.grid-container {display: grid;} Grid Turns an element into a block-level grid. Grids default to 1 column.
.grid-container {display: inline-grid;} Inline Grid Turns an element into an inline grid.
.grid-container {grid-template-rows: 20% 20% 600px;} Grid Template Rows Defines the number of rows in the grid, and the height of each row.
.grid-container {grid-template-columns: 100px 200px;} Grid Template Columns Defines the number of columns in the grid, and the width of each column. The example has 2 columns, where the first has a width of 100px, and the second has a width of 200px. You can also define the size of the columns as a percentage of the entire grid's width (e.g. 20%).
.grid-container {grid-template: 200px 300px / 20% 10% 70%;} Grid Template Can be used in place of grid-template-columns and grid-template-rows. When using grid-template, the values before the slash will determine the size of each row. The values after the slash determine the size of each column.
.grid-container {grid-template: 2fr 1fr 1fr / 1fr 3fr 1fr;} Fraction A relative sizing unit. This unit was specifically created for use in CSS Grid. Using fr makes it easier to prevent grid items from overflowing the boundaries of the grid. It is possible to use fr with other units as well. When this happens, each fr represents a fraction of the available space (e.g. 1fr 60px 1fr).
.grid-container {grid-template-columns: repeat(3, 100px);} Repeat The properties that define the number of rows and columns in a grid can take a function as a value. repeat() is one of these functions. The repeat function will duplicate the specifications for rows or columns a given number of times. In the example above, using the repeat function will make the grid have three columns that are each 100 pixels wide. Repeat is particularly useful with fr. For example, repeat(5, 1fr) would split your table into five equal rows or columns. Finally, the second parameter of repeat() can have multiple values (e.g. grid-template-columns: repeat(2, 20px 50px)). This code will create four columns where the first and third columns will be 20 pixels wide and the second and fourth will be 50 pixels wide.
.grid-container {grid-template-columns: 100px minmax(100px, 500px) 100px;} Min Max Defines a minimum and maximum width or height for a column or row, respectively.
.grid-container {gap: 20px 10px;} Grid Gap The CSS properties row-gap: and column-gap: will put blank space between every row and column in the grid. It is important to note that grid gap properties do not add space at the beginning or end of the grid. A shorthand CSS property, gap: can set the row and column gap at the same time (the first number is the row gap, the second number is the column gap). If only one value is given, it will set the column gap and the row gap to that value.
.grid-item {grid-row-start: 1; grid-row-end: 3;} Multiple Row Items Can make single grid items take up multiple rows. The values that grid-row-start: and grid-row-end: accept are grid lines. The value for grid-row-start: should be the row at which you want the grid item to begin. The value for grid-row-end: should be one greater than the row at which you want the grid item to end. It is possible for the value of grid-row-start to be greater than that of grid-row-end. Both properties can also each have negative values. When an item spans multiple rows or columns using these properties, it will also include the gap if any exists. For example, if an item spans two rows of height 100 pixels and there is a ten-pixel gap, then the item will have a total height of 210 pixels.
.grid-item {grid-row: 4 / 6;} Grid Row Shorthand for grid-row-start: and grid-row-end:. The starting row goes before the “/“ and the ending row goes after it.
.grid-item {grid-column: 4 / span 2} Grid Column Shorthand for grid-column-start: and grid-column-end:. The starting row goes before the “/“ and the ending row goes after it. When using these properties, we can use the keyword span to start or end a column or row, relative to its other end. The example is telling the item element to begin in column four and take up two columns of space (columns 4 and 5). If you know where you want your grid item to start and how long it should be, use span!
.grid-item {grid-area: 2 / 3 / 4 / span 5;} Grid Area Shorthand for grid-row: and grid-column:. grid-area: takes four values separated by slashes: grid-row-start: / grid-column-start: / grid-row-end: / grid-column-end:. Using grid-area: is an easy way to place items exactly where you want them in a grid.
.grid-container {grid-template-areas: "header header" "nav nav" "info services" "footer footer";} header {grid-area: header;} Grid Template Areas Allows you to name sections of your web page to use as values in the grid-row-start, grid-row-end, grid-column-start,grid-column-end, and grid-area properties. This property is declared on grid containers. The example creates a 2-column, 4-row layout. In each ruleset below .container, we use the grid-area property to tell that section to cover the portion of the page specified. In the example, the header element spans the first row and both columns.
Overlapping Elements Using grid-area: on different elements, the rows and columns may sometimes overlap. Using a z-index: can be helpful.
.grid-container {justify-items: center;} Justify Items There are two axes in a grid layout — the column (or block) axis and the row (or inline) axis. justify-items: is a property that positions grid items along the inline, or row, axis. It is declared on grid containers. Includes: start; — aligns grid items to the left side of the grid area, end; aligns grid items to the right side of the grid area, center; aligns grid items to the center of the grid area, stretch; stretches all items to fill the grid area.
.grid-container {justify-content: space-around;} Justify Content justify-content: positions the entire grid column along the row axis. This property is declared on grid containers. Includes: start; — aligns the grid to the left side of the grid container, end; — aligns the grid to the right side of the grid container, center; — centers the grid horizontally in the grid container, stretch; — stretches the grid items to increase the size of the grid to expand horizontally across the container, space-around; — includes an equal amount of space on each side of a grid element, resulting in double the amount of space between elements as there is before the first and after the last element, space-between; — includes an equal amount of space between grid items and no space at either end, space-evenly; — places an even amount of space between grid items and at either end.
.grid-container {align-items: center;} Align Items Positions grid items along the block, or column axis. This means that it positions items from top to bottom. This property is declared on grid containers. Includes: start; — aligns grid items to the top side of the grid area, end; — aligns grid items to the bottom side of the grid area, center; — aligns grid items to the center of the grid area, stretch; — stretches all items to fill the grid area.
.grid-container {align-content: space-evenly;} Align Content Positions grid rows along the column axis, or from top to bottom, and is declared on grid containers. Includes: start; — aligns the grid to the top of the grid container, end; — aligns the grid to the bottom of the grid container, center; — centers the grid vertically in the grid container, stretch; — stretches the grid items to increase the size of the grid to expand vertically across the container, space-around; — includes an equal amount of space on each side of a grid element, resulting in double the amount of space between elements as there is before the first and after the last element, space-between; — includes an equal amount of space between grid items and no space at either end, space-evenly; — places an even amount of space between grid items and at either end.
.grid-item {justify-self: } Justify Self Specifies how an individual element should position itself with respect to the row axis. This property will override justify-items: for any item on which it is declared. Declared on grid items. Includes: start; — positions grid items on the left side of the grid area, end; — positions grid items on the right side of the grid area, center; — positions grid items on the center of the grid area, stretch; — positions grid items to fill the grid area (default).
.grid-item {align-self: } Align Self Specifies how an individual element should position itself with respect to the column axis. This property will override align-items: for any item on which it is declared. Declared on grid items. Includes: start; — positions grid items on the top of the grid area, end; — positions grid items on the bottom of the grid area, center; — positions grid items on the center of the grid area, stretch; — positions grid items to fill the grid area (default).
Implicit vs. Explicit Grid What happens if the developer has specified a 3-column, 5-row grid (for a total of 15 items), but the search results return 30 items? The implicit grid takes over. The implicit grid is an algorithm built into the specification for CSS Grid that determines default behavior for the placement of elements when there are more than fit into the grid specified by the CSS. The default behavior of the implicit grid is as follows: items fill up rows first, adding new rows as necessary. New grid rows will only be tall enough to contain the content within them.
.grid-container {grid-auto-rows: } Grid Auto Rows Specifies the height of implicitly added grid rows. Accepts the same values as its explicit counterpart: grid-template-rows:. Can apply the following: px, %, fr, repeat(). If we did not specify grid-auto-rows:, the rows would be auto-adjusted to the height of the content of the grid items.
.grid-container {grid-auto-columns: } Grid Auto Columns Specifies the width of implicitly added grid columns. Accepts the same values as its explicit counterpart: grid-template-columns:. Can apply the following: px, %, fr, repeat().
.grid-container {grid-auto-flow: row;} Grid Auto Flow Specifies whether new elements should be added to rows or columns, and is declared on grid containers. Includes: row; — specifies the new elements should fill rows from left to right and create new rows when there are too many elements (default), column; — specifies the new elements should fill columns from top to bottom and create new columns when there are too many elements, dense; — this keyword invokes an algorithm that attempts to fill holes earlier in the grid layout if smaller elements are added. You can pair row; or column; with dense;, like this: grid-auto-flow: row dense;.
Flexbox
Example Name Description
div.container {display: flex;} Display Flex Designates an element as a flex container. Flex containers are helpful tools for creating websites that respond to changes in screen sizes. Child elements of flex containers will change size and location in response to the size and position of their parent container. In the example, all divs with the class "container" are flex containers, and their child elements are flex items. This is a block-level declaration. Child elements will appear inline.
div.container {display: inline-flex;} Inline Flex Creates flex containers that appear inline. If the parent element is too small, the flex items will shrink to accommodate the parent container’s size.
div.container {justify-content: flex-end;} Justify Content Aligns flex items horizontally. Includes: flex-start; all items will be positioned in order, starting from the left of the parent container, with no extra space between or before them. flex-end; all items will be positioned in order, with the last item starting on the right side of the parent container, with no extra space between or after them. center; all items will be positioned in order, in the center of the parent container with no extra space before, between, or after them. space-around; items will be positioned with equal space before and after each item, resulting in double the space between elements. space-between; items will be positioned with equal space between them, but no extra space before the first or after the last elements.
div.container {align-items: baseline;} Align Items Aligns flex items vertically. Includes: flex-start; all elements will be positioned at the top of the parent container. flex-end; all elements will be positioned at the bottom of the parent container. center; the center of all elements will be positioned halfway between the top and bottom of the parent container. baseline; the bottom of the content of all items will be aligned with each other. stretch; if possible, the items will stretch from top to bottom of the container (this is the default value; elements with a specified height will not stretch; elements with a minimum height or no height specified will stretch).
.flexitem {flex-grow: 1;} Flex Grow Allows us to specify if items should grow to fill a container and also which items should grow proportionally more or less than others (value=1, 2, 3, etc.). The default value of flex-grow is 0. Setting a max-width sets a limit to how much an element will expand. Margins are unaffected by flex-grow.
.flexitem {flex-shrink: 2;} Flex Shrink Can be used to specify which elements will shrink and in what proportions. The default value of flex-shrink is 1. Margins are unaffected by flex-shrink. Will only be employed if the parent container is too small or the browser is adjusted.
.flexitem {flex-basis: 100px;} Flex Basis Another way of specifying the width of a flex item. flex-basis allows us to specify the width of an item before it stretches or shrinks.
.flexitem {flex: 2 1 150px;} Flex Provides a convenient way for specifying how elements stretch and shrink, while simplifying the CSS required. Allows you to declare flex-grow, flex-shrink, and flex-basis all in one line (in that order). You can set flex-grow and flex-basis by using two values (1 100px;), or flex-grow and flex-shrink by using two values (2 1), but not flex-shrink and flex-basis.
div.container {flex-wrap: wrap;} Flex Wrap Sometimes, we don’t want our content to shrink to fit its container. Instead, we might want flex items to move to the next line when necessary. Includes: wrap; child elements of a flex container that don’t fit into a row will move down to the next line. wrap-reverse; the same functionality as wrap, but the order of rows within a flex container is reversed. nowrap; prevents items from wrapping; this is the default value and is only necessary to override a wrap value set by a different CSS rule.
div.container {align-content: space-around;} Align Content For aligning elements within a single row. If a flex container has multiple rows of content, we can use align-content to space the rows from top to bottom. Includes: flex-start; all rows of elements will be positioned at the top of the parent container with no extra space between. flex-end; all rows of elements will be positioned at the bottom of the parent container with no extra space between. center; all rows of elements will be positioned at the center of the parent element with no extra space between. space-between; all rows of elements will be spaced evenly from the top to the bottom of the container with no space above the first or below the last. space-around; all rows of elements will be spaced evenly from the top to the bottom of the container with the same amount of space at the top and bottom and between each element. stretch; if a minimum height or no height is specified, the rows of elements will stretch to fill the parent container from top to bottom (default value).
div.container {flex-direction: column;} Flex Direction Flex containers have two axes: a main axis and a cross axis. By default, the main axis is horizontal and the cross axis is vertical. The main axis is used to position flex items with the following properties: justify-content: flex-wrap: flex-grow: flex-shrink:. The cross axis is used to position flex items with the following properties: align-items: align-content:. The main axis and cross axis are interchangeable. We can switch them using the flex-direction property. If we add the flex-direction property and give it a value of column, the flex items will be ordered vertically, not horizontally. The flex-direction: property can accept four values: row; elements will be positioned from left to right across the parent element starting from the top left corner (default). row-reverse; elements will be positioned from right to left across the parent element starting from the top right corner. column; elements will be positioned from top to bottom of the parent element starting from the top left corner. column-reverse; elements will be positioned from the bottom to the top of the parent element starting from the bottom left corner.
div.container {flex-flow: column wrap;} Flex Flow Like the shorthand flex property, the shorthand flex-flow property is used to declare both the flex-wrap and flex-direction properties in one line.
Nested Flexboxes It is also possible to position flex containers inside of one another.

Relative Measurements With CSS, you can avoid hard coded measurements and use relative measurements instead. Relative measurements offer an advantage over hard coded measurements, as they allow for the proportions of a website to remain intact regardless of screen size or layout.
.heading {font-size: 2em;} Em Incorporating relative sizing starts by using units other than pixels. One unit of measurement you can use in CSS to create relatively-sized content is the em. Today, the em represents the font-size of the current element or the default base font-size set by the browser if none is given. For example, if the base font of a browser is 16 pixels (which is normally the default size of text in a browser), then 1 em is equal to 16 pixels. 2 ems would equal 32 pixels, and so on.
h1 {font-size: 2rem;} Rem Rem stands for root em. It acts similar to em, but instead of checking parent elements to size font, it checks the root element. The root element is the <html> tag. Most browsers set the font size of <html> to 16 pixels, so by default rem measurements will be compared to that value. To set a different font size for the root element, you can add a CSS rule. One advantage of using rem is that all elements are compared to the same font size value, making it easy to predict how large or small font will appear. If you are interested in sizing elements consistently across an entire website, the rem measurement is the best unit for the job. If you’re interested in sizing elements in comparison to other elements nearby, then the em unit would be better suited for the job.
#main-nav {height: 100vh;} View Width / View Height View width. 1vw is 1% of the width of the viewport. View height. 1vh is 1% of the height of the viewport.
.main {height: 300px; width: 500px;} .main .subsection {height: 50%; width: 50%;} Percentages To size non-text HTML elements relative to their parent elements on the page you can use percentages. Percentages are often used to size box-model values, like width and height, padding, border, and margins. When height and width are set using percentages, the dimensions of child elements are calculated based on the dimensions of the parent element. When percentages are used to set padding and margin, however, they are calculated based only on the width of the parent element. They can also be used to set positioning properties (top, bottom, left, right). Be careful, a child element’s dimensions (.subsection) may be set erroneously if the dimensions of its parent element (.main) aren’t set first. Because the box model includes padding, borders, and margins, setting an element’s width to 100% may cause content to overflow its parent container. While tempting, 100% should only be used when content will not have padding, border, or margin.
When to Use Each When using relative sizing, ems and rems should be used to size text and dimensions on the page related to text size (i.e. padding around text). This creates a consistent layout based on text size. Otherwise, percentages should be used.
p {min-width: 300px; max-width: 600px;} Min / Max Width Although relative measurements provide consistent layouts across devices of different screen sizes, elements on a website can lose their integrity when they become too small or large. The unit of pixels is used to ensure hard limits on the dimensions of the element(s).
p {min-height: 150px; max-height: 300px;} Min / Max Height Although relative measurements provide consistent layouts across devices of different screen sizes, elements on a website can lose their integrity when they become too small or large. The unit of pixels is used to ensure hard limits on the dimensions of the element(s).
.container img {max-width: 100%; height: auto; display: block;} Scaling Images & Video The example ensures that images scale with the width of the container. The height property is set to auto, meaning an image’s height will automatically scale proportionally with the width. Finally, the last line will display images as block level elements (rather than inline-block, their default state). This will prevent images from attempting to align with other content on the page (like text), which can add unintended margin to the images.
body {background-image: url('#'); background-repeat: no-repeat; background-position: center; background-size: cover;} Scaling Background Images In the example, the first CSS declaration sets the background image (# is a placeholder for an image URL in this example). The second declaration instructs the CSS compiler to not repeat the image (by default, images will repeat). The third declaration centers the image within the element. The final declaration, however, is the focus of the example above. It’s what scales the background image. The image will cover the entire background of the element, all while keeping the image in proportion. If the dimensions of the image exceed the dimensions of the container then only a portion of the image will display.
Media Queries
Example Name Description
Viewport The total viewable area for a user; this area varies depending on the device. The viewport is smaller on a mobile device and larger on a desktop.
<head> ... <meta name="viewport" content="width=device-width, initial-scale=1"> ... </head> HTML Viewport Meta Tag Based on the size of the viewport, the meta tag (<meta>) is used to instruct the browser on how to render the webpage’s scale and dimensions. For instance, say if a web page is 960px and the device is 320px wide. Adding the viewport meta tag will squish the content down until it is 320px — there is no need for the user to zoom out and view the whole page! The name="viewport" attribute: tells the browser to display the web page at the same width as its screen. The content attribute: defines the values for the <meta> tag, including width and initial-scale. The width=device-width key-value pair: controls the size of the viewport in which it sets the width of the viewport to equal the width of the device. The initial-scale=1 key-value pair: sets the initial zoom level.
@media only screen and (max-width: 480px) { body { font-size: 12px; } } Media Queries CSS uses media queries to adapt a website’s content to different screen sizes. With media queries, CSS can detect the size of the current screen and apply different CSS styles depending on the width of the screen. In the example, the media query defines a rule for screens smaller than 480 pixels (approximately the width of many smartphones in landscape orientation). @media — This keyword begins a media query rule and instructs the CSS compiler on how to parse the rest of the rule. only screen — Indicates what types of devices should use this rule. In early attempts to target different devices, CSS incorporated different media types (screen, print, handheld). The rationale was that by knowing the media type, the proper CSS rules could be applied. However, “handheld” and “screen” devices began to occupy a much wider range of sizes and having only one CSS rule per media device was not sufficient. screen is the media type always used for displaying content, no matter the type of device. The only keyword is added to indicate that this rule only applies to one media type (screen). and (max-width : 480px) — This part of the rule is called a media feature, and instructs the CSS compiler to apply the CSS styles to devices with a width of 480 pixels or smaller. Media features are the conditions that must be met in order to render the CSS within a media query. CSS rules are nested inside of the media query’s curly braces {}. The rules will be applied when the media query is met. In the example above, the text in the body element is set to a font-size of 12px when the user’s screen is less than 480px.
@media only screen and (min-width: 320px) and (max-width: 480px) {} Range Specific screen sizes can be targeted by setting multiple width and height media features, in the form of min-width: min-height: max-width: and max-height:. In the example, the word and allows us to chain multiple requirements together.
@media only screen and (min-resolution: 300dpi) {} Resolution Targets screen resolution. Many times we will want to supply higher quality media (images, video, etc.) only to users with screens that can support high resolution media. Targeting screen resolution also helps users avoid downloading high resolution (large file size) images that their screen may not be able to properly display. To target by resolution, we can use the min-resolution: and max-resolution: media features. These media features accept a resolution value in either dots per inch (dpi) or dots per centimeter (dpc). High-resolution screens typically have a DPI greater than ~150.
@media only screen and (max-width: 480px) and (min-resolution: 300dpi) {} And Operator You can chain multiple media features of the same type in one media query by using the and operator. The and operator can be used to require multiple media features.
@media only screen and (min-width: 480px), (orientation: landscape) {} Comma Separated List If only one of multiple media features in a media query must be met, media features can be separated in a comma separated list. The example requires only one of the media features to be true for its CSS to apply.
@media only screen and (min-width: 480px), (orientation: landscape) {} Orientation The orientation media feature detects if the page has more width than height. If a page is wider, it’s considered landscape, and if a page is taller, it’s considered portrait.
@media only screen and (min-width: 768px) and (max-width: 1024px) and (orientation: landscape) {} Breakpoints The points / values at which media queries are set are called breakpoints. Breakpoints are the screen sizes at which your web page does not appear properly. The example creates a screen size range the size of a tablet in landscape mode and also identifies the orientation. Rather than set breakpoints based on specific devices, the best practice is to resize your browser to view where the website naturally breaks based on its content. The dimensions at which the layout breaks or looks odd become your media query breakpoints. Within those breakpoints, we can adjust the CSS to make the page resize and reorganize. Common breakpoints include: 480px (mobile), 768px (tablet), 1024px (tablet landscape), 1600px (laptop), >1600px (desktop).
