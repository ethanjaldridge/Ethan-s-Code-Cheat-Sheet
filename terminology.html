Term Definition
Front-End The part of a web app that runs in a user's browser.
Back-End The part of a web app that runs on the web server.
Full-Stack The "stack" of coding languages that a developer has command over. A full stack developer is capable of both front-end and back-end development. A full stack developer can build a web application from end to end.
Version Control Allows a team of developers to write code on the same project without overwriting existing code (Git).
API (Application Programming Interface) A set of defined rules that allow different applications to communicate with each other.
Stack Refers to a developer's tech stack: programming languages, libraries, frameworks, APIs, and other tools.
ARPANET (Advanced Research Projects Agency Network) In 1969, the U.S. D.O.D. funded the creation of ARPANET, which connected supercomputing centers run by government agencies and universities.
TCP/IP (Transmission Control Protocol and Internet Protocol) In the 1970's, TCP/IP was created to provide standards for the transfer of data. Eventually, its adoption led to the creation of the internet.
Internet Network of connected computing devices (began in 1980's).
World-wide web (WWW) Invented by Tim Berners-Lee in 1989, the world wide web is a collection of interlinked websites and other web resources.
Client-server model A way of viewing the dialogue between users and data on the internet.
Client The user's device or program that is making a request for data (browser, application, etc.).
Server The device or program in a network that waits for inbound requests and sends back data (in-house, rented at a data center, cloud server). A specialized computer dedicated to a network.
HTTP (HyperText Transfer Protocol) Browsers and servers communicate with each other using the HTTP protocol. The “s” in “https” refers to the secure nature of the data transfer. An HTTP request seeks to retrieve a file written in a given code (e.g. HTML). In most modern browsers, requests are made in parallel (at the same time). Information is displayed as soon as possible.
Get request Retrieve information from the given server.
Post request Send data to the server (e.g. file upload).
Put Replace current data with uploaded content.
Delete Removes current data.
HTTP status code Indicates whether the HTTP request was completed. If there was an error, some information about the type of error may be provided. 200 (OK) - the request has succeeded. 301 (Moved Permanently) - the resource has been moved, and the client is being redirected. 404 (Not Found) - requested data was not found. 500 (Internal Server Error) - the server encountered an unexpected error.
HTML (HyperText Markup Language) HTML files hold the content of a website, as well as any links for additional assets or code that are needed to display the website properly.
Markup Language A way of annotating text that is distinguishable from the text itself.
Hypertext Text that is linked to other text.
Attribute Provides additional information about an HTML element (name=”value”).
HTML link tag Connects HTML file and CSS file to apply style to HTML element(s).
CSS (Cascading Style Sheets) Creates the style and layout of a webpage. When the browser receives the CSS style sheet from the server, it applies the visual styles to the content of the website.
Selectors In CSS, these allow you to specify the HTML elements that a style should be applied to.
Visual rules In CSS, how the content should be displayed.
JavaScript Makes the webpage interactive; the “behavior” of the webpage. In 1995, a NetScape employee, Brendan Eich, was given ten days to help build the company a lightweight scripting language. As a result, Eich built Mocha, which was later renamed JavaScript. The moral of the story – JavaScript was never intended to become the standard language for the web! JavaScript is desirable because: it can be used for front and back-end development, it is standardized, so it's constantly being updated, and it creates interactivity while allowing developers to import libraries and frameworks to change functionality.
Static webpage A webpage that doesn't use JavaScript.
Website assets Images and videos.
Loading An indication of the delay in the request-response cycle between the browser and server. The speed at which a webpage loads is determined by the speed of the user's connection, the size of the website, and the physical distance between the browser and the server.
Web 2.0 A cluster of web applications in the early 2000's that improved the static websites of the years prior. They are characterized by: offering content that responds to user input without reloading the webpage, and emphasizing user-generated content and sharing (social media, blogs, wiki).
jQuery A JavaScript framework that allows data retrieval while the webpage is running.
Web Frameworks (that connect to databases) Enable user-generated content to be created, stored, and displayed. Examples include Spring, Django, and Ruby-on-Rails.
Swift A programming language used for iOS to develop mobile applications (JavaScript is also commonly used).
Separation of concerns Allows large codebases to be maintained when each section has clearly defined problems that it is trying to solve.
Functions Reusable blocks of code designed to perform specific functions.
Events Allows JavaScript to respond to user behaviors.
SQL Structured Query Language. Used by Data Scientists to analyze data. Web developers and software developers also use SQL to build apps that can save, modify, and access data.
Query A program that retrieves data from a table.
Programming Giving a set of instructions to a computer to execute; the mental process of thinking up instructions to give to a machine.
Coding The process of transforming ideas into a written language that a computer can understand.
URL Uniform Resource Locator; a path to a specific file.
Metadata Information about the page that isn't displayed directly on the web page.
Mozilla Developer Network (MDN) An open source network of documentation and tools pertaining to web development:
Font Refers to the technical term typeface, or font family.
Shorthand property A declaration that uses multiple properties as values.
Boilerplate The introductory lines of code that allow a browser to read it.
Integrated Development Environments (IDEs) Edit, compile, and debug your code through one application or interface.
DOM Document Object Model.
Directory A folder that holds files.
Root Directory The first directory.
String Sequence of characters treated as a single piece of data (a word).
Argument When a file, directory, or program is passed into a command.
Environment variables Variables that can be used across commands and programs and hold information about the environment.
Skeuomorphism The concept of UI elements that replicate or imitate real-life counterparts is known as skeuomorphism.
Flat Design An alternative approach to skeuomorphism that embraces the fact that computer interfaces do not necessarily need to model real-life interfaces.
Affordance The potential for interaction with an object.
Signifier Aspect of an object that a designer uses to indicate potential and intended affordances of an object.
User experience (UX) patterns Establish reusable solutions to common problems.
Hero section The area on a website that immediately shows up below the logo and menu. The hero section should include: what you offer, why people should trust you, the benefits of working with you, and what actions they should take.
Wireframes BLUEPRINT. Communicates how a designer intends to layout a product; the structure (how pieces of a webpage are put together), content (what will be displayed on the page), and functionality (how it will work). Emphasizes usability over aesthetics. Allows developers to provide feedback on the design. Methods include: sketching, UX design software (Wires, and Adobe XD), and graphic design software (Adobe Photoshop, Illustrator).
Prototypes Exploring a design by building a low-high fidelity simulation of user-interface interaction. Good for: pitching an idea, making a sale, better client-dev communication, and finding problems. Tools include: presentation software (PowerPoint, Keynote), design software, and native prototype (writing code).
Hosting Renting a server from a hosting provider to share your application/website with the world. Types include: website builders (Wix, Squarespace; good to have a website without building it yourself), shared web hosting (sharing a server with others), dedicated (you get your own server; ideal), and cloud hosting (running your app in a network of data centers; good for running different parts of your app on different machines).
Cloud Hosting Includes: Infrastructure as a Service (IaaS); raw infrastructure resources, cloud provider gives you access to (servers, storage, networks), you have to maintain them (apps, data, middleware, runtime, OS). Platforms as a Service (PaaS); cloud provider takes care of servers and gives you a way to run your app, you manage (apps, data). Functions as a Service (FaaS); you give cloud provider specific functions and they execute them, you are responsible for (functions).
Good Options for Hosting GitHub Pages; good for static applications (HTML, CSS, React), Heroku; good for full-stack apps (you have a node.Js server and SQL database), DigitalOcean; full control over cloud resources (cloud provider).
GitHub Allows you to store files and code for a website in a repository.
GitHub Pages Generates a personal URL to share a website.
NetScape Microsoft (Internet Explorer) and Netscape (NetScape Navigator) competed for dominance in the early internet market.
Electron.js Excellent for making desktop applications that need to work across different devices regardless of operating system.
ECMAScript In 1997, Ecma International released ECMA-262 which sets standards for the first version of a scripting language called ECMAScript, shortened to ES. These new ECMAScript standards provided rules for the architecture of JavaScript features. To fully distinguish the difference between JavaScript and ECMAScript: if you want to create an app or program you can use JavaScript — if you want to create a new scripting language you can follow the guidelines in ECMAScript. So, when you see ES6 or JavaScript ES6, it means that that version of JavaScript is following the specifications in the sixth edition of ECMAScript!
Site Reliability Engineer Finds and reports errors in web platforms.
Test Engineer Builds automated tests to discover errors in software and make sure that it meets a company’s standards.
How to Debug Code Errors The error will likely give you relevant information in the first couple lines...

TypeError: students.filter is not a function
  at /home/runner/FearlessNewDev/index.js:21:26

The first line says that the JavaScript engine doesn’t think that .filter() is a function — but it should be, so there might be an issue in how it’s being used. The second line points to the line of code that is throwing the error, which is line 21 and at the 26th character.
Luhn algorithm The Luhn algorithm or Luhn formula, also known as the "modulus 10" or "mod 10" algorithm, named after its creator, IBM scientist Hans Peter Luhn, is a simple check digit formula used to validate a variety of identification numbers (i.e. credit card numbers).

  1. Starting from the farthest digit to the right, AKA the check digit, iterate to the left.
  2. As you iterate to the left, every other digit is doubled, starting with the digit to the left of the check digit (the check digit is not doubled). If the number is greater than 9 after doubling, subtract 9 from its value.
  3. Sum up all the digits in the (new) credit card number.
  4. If the sum modulo 10 is 0 (if the sum divided by 10 has a remainder of 0; sum % 10) then the number is valid, otherwise, it’s invalid.
